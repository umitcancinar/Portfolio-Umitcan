## 🔄 İş Akışı ve Dağıtım (Workflow)

Proje, sürekli entegrasyon ve dağıtım (CI/CD) mantığıyla yönetilmektedir:

1.  **Geliştirme (Local Dev):** Kod değişiklikleri yerel ortamda yapılır ve test edilir.
2.  **Dağıtım (Deploy):** `npm run deploy` komutu ile proje derlenir (build alınır).
3.  **Yayınlama (Hosting):** Derlenen dosyalar otomatik olarak GitHub Pages'in `gh-pages` dalına yüklenir.
4.  **DNS & SSL:** Namecheap üzerindeki CNAME kayıtları trafiği GitHub'a yönlendirir ve GitHub tarafından sağlanan SSL sertifikası ile site güvenli (`https://`) olarak yayınlanır.

## 💻 Kurulum ve Yerel Çalıştırma

### 📋 Ön Koşullar

- [Node.js](https://nodejs.org/) v18+ ve npm yüklü olmalı
- [Neon PostgreSQL](https://neon.tech/) hesabı (veya herhangi bir PostgreSQL veritabanı)
- Git

### 🚀 Adım Adım Kurulum

#### 1. Repoyu Klonlayın

```bash
git clone https://github.com/umitcancinar/Portfolio-Umitcan.git
cd Portfolio-Umitcan
```

#### 2. Frontend Bağımlılıklarını Yükleyin

```bash
npm install
```

#### 3. Backend Bağımlılıklarını Yükleyin

```bash
cd backend
npm install
cd ..
```

#### 4. Backend Ortam Değişkenlerini Ayarlayın

`backend/.env` dosyası oluşturun:

```env
NODE_ENV=development
PORT=5000
CLIENT_URL=http://localhost:3000
DATABASE_URL="postgresql://user:pass@ep-xxxx.us-east-2.aws.neon.tech/portfolio?sslmode=require"
JWT_SECRET=your-strong-jwt-secret-key
JWT_EXPIRES_IN=7d
ADMIN_EMAIL=admin@umitcancinar.me
ADMIN_PASSWORD=your-secure-password
```

#### 5. Veritabanını Oluşturun ve Seed'leyin

```bash
cd backend
npx prisma generate
npx prisma migrate dev --name init
npm run db:seed
npx prisma studio  # isteğe bağlı
```

#### 6. Frontend Ortam Değişkenlerini Ayarlayın

Proje kök dizininde `.env` dosyası oluşturun:

```env
REACT_APP_API_BASE_URL=http://localhost:5000/api
```

#### 7. Backend'i Başlatın

```bash
cd backend
npm run dev
# → Backend http://localhost:5000 adresinde çalışır
```

#### 8. Frontend'i Başlatın (yeni terminal penceresinde)

```bash
# Proje kök dizininde
npm start
# → Frontend http://localhost:3000 adresinde çalışır
```

### 📦 Dağıtım (Deploy)

Frontend'i GitHub Pages'e dağıtmak için:

```bash
npm run deploy
```

Bu komut:
1. Projeyi derler (`npm run build`)
2. `build/` klasörünü `gh-pages` branch'ine yükler
3. GitHub Pages otomatik olarak güncellenir

## 👑 Admin Panel Kullanım Kılavuzu

Admin paneli, portfolyo içeriğinizi yönetmek için güçlü bir arayüz sunar.

### 🔐 Giriş Yapma

1. Tarayıcınızda `https://www.umitcancinar.me/login` adresine gidin (yerelde `http://localhost:3000/login`)
2. Admin e-posta adresinizi ve şifrenizi girin
3. Başarılı girişten sonra otomatik olarak `/admin` paneline yönlendirilirsiniz

### 📊 Dashboard

Admin paneli iki ana sekmeden oluşur:

#### 📁 Projeler Sekmesi

**Yeni Proje Ekleme:**
1. "Proje Ekle" formunu doldurun:
   - **Başlık** (zorunlu) — Proje adı
   - **Açıklama** (zorunlu) — Kısa proje özeti
   - **Teknolojiler** — Virgülle ayırarak (örn: `React, Node.js, PostgreSQL`)
   - **Görsel URL** — Proje kapak görseli linki
   - **GitHub** — Proje GitHub repo linki
   - **Demo** — Canlı demo linki
2. "Ekle" butonuna tıklayın
3. Proje listenizde yeni projeniz görünecektir

**Proje Silme:**
- Proje listesinde "Sil" butonuna tıklayın ve onaylayın

#### ✍️ Blog Sekmesi

**Yeni Blog Yazısı Ekleme:**
1. "Yeni Yazı" formunu doldurun:
   - **Başlık** (zorunlu) — Yazı başlığı
   - **Slug** — Otomatik oluşturulur, elle de düzenlenebilir
   - **Kategori** — `Genel`, `Teknoloji`, `Yazılım`, `Eğitim`
   - **Özet** — Kısa açıklama
   - **İçerik** (zorunlu) — Yazının tam metni
   - **Kapak Görseli** — Blog kapak resmi URL'si
   - **Etiketler** — Virgülle ayırarak (örn: `react, nodejs, javascript`)
2. "Yayınla" butonuna tıklayın

**Blog Yazısı Düzenleme:**
- Yazılar listesinde "Düzenle" butonuna tıklayın
- Form otomatik olarak mevcut verilerle doldurulur
- Değişiklikleri yapıp "Güncelle" butonuna tıklayın
- "İptal" ile düzenleme modundan çıkabilirsiniz

**Blog Yazısı Silme:**
- Yazılar listesinde "Sil" butonuna tıklayın ve onaylayın

### 🔒 Güvenlik

- Admin paneline sadece JWT token ile yetkilendirilmiş kullanıcılar erişebilir
- Oturum süresi boyunca token tarayıcıda localStorage'da saklanır
- Çıkış yapıldığında token otomatik temizlenir
- Tüm admin API istekleri `Authorization: Bearer <token>` header'ı ile korunur

## ⚡ Performans Özellikleri

- **Code Splitting:** Tüm sayfalar `React.lazy()` ile ayrı bundle'lara bölünmüştür
- **memo():** Gereksiz render'ları önlemek için bileşenler memoize edilmiştir
- **Optimized Particles:** `@tsparticles/slim` ile hafif partikül motoru, 60 FPS hedefli
- **Lazy Loading:** Footer, ContactFloating gibi bileşenler sadece ihtiyaç anında yüklenir
- **SEO Optimizasyonu:** Her sayfa için dinamik meta etiketleri (React Helmet Async)
- **Responsive Tasarım:** Mobil, tablet ve masaüstü için optimize edilmiş CSS
- **Rate Limiting:** API istekleri rate limit ile korunur (DDOS önlemi)
- **Helmet Güvenlik:** HTTP güvenlik başlıkları ile koruma
